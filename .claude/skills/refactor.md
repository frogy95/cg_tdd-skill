# /refactor — 리팩토링 스킬

## 역할
TDD 사이클의 **Refactor 단계**: 테스트를 안전망 삼아 코드 품질을 개선한다.
**동작은 변경하지 않는다** — 테스트가 계속 통과하면 올바른 리팩토링이다.

## 규칙
- **테스트를 통과하는 상태를 항상 유지**한다
- **새로운 기능을 추가하지 않는다**
- 리팩토링 전/후 테스트 결과가 동일해야 한다
- 작은 단계로 나누어 진행하며, 각 단계마다 테스트를 실행한다

## 리팩토링 집중 영역

### 코드 품질
- 중복 제거 (DRY 원칙)
- 긴 함수 분리
- 복잡한 조건문 단순화

### 가독성
- 변수명/함수명 명확화
- 의도를 드러내는 코드로 개선
- 불필요한 주석 제거 (코드 자체로 설명되도록)

### 구조
- 적절한 추상화 도입
- 책임 분리 (단일 책임 원칙)
- 의존성 정리

## 실행 절차

1. 현재 테스트가 모두 통과하는지 확인한다
2. 개선할 부분을 파악하고 **작은 단위로** 리팩토링한다
3. 각 변경 후 테스트를 실행하여 통과를 확인한다
4. 모든 개선이 완료되면 최종 테스트를 실행한다

## 완료 신호
리팩토링 완료 후:
```
✨ Refactor 완료
개선 사항: [주요 변경 내용 목록]
테스트: 모두 통과 ✅
다음 단계: /verify — 전수 검증을 실행합니다
```

# /red — 실패 테스트 작성 스킬

## 역할
TDD 사이클의 **Red 단계**: 명세를 기반으로 실패하는 테스트를 작성한다.
테스트가 실패하는 것이 이 단계의 **성공**이다.

## 규칙
- **구현 코드(소스 파일)는 절대 수정하지 않는다**
- **테스트 파일만** 생성하거나 수정한다
- 테스트는 현재 실패해야 정상이다 — 통과하면 테스트가 잘못된 것
- `specs/<feature-name>.md`의 완료 기준을 테스트 케이스로 변환한다

## 실행 절차

1. `specs/<feature-name>.md`를 읽어 완료 기준을 파악한다
2. 각 완료 기준과 엣지 케이스를 테스트 케이스로 변환한다
3. 테스트 파일을 작성한다 (구현 코드 수정 없음)
4. 테스트를 실행하여 **실패를 확인**한다

## 테스트 작성 원칙

```
// 좋은 테스트: 의도가 명확하다
it('이메일 없이 회원가입 시 오류를 반환한다', () => { ... })

// 나쁜 테스트: 구현 세부사항에 의존한다
it('validateEmail 함수가 false를 반환한다', () => { ... })
```

- 테스트명은 **동작과 기대 결과**를 서술한다
- 하나의 테스트는 하나의 동작만 검증한다
- Arrange-Act-Assert 패턴을 따른다

## 완료 신호
테스트 실행 후 실패를 확인하면:
```
🔴 Red 완료: <테스트 파일 경로>
실패한 테스트: N개 (예상된 실패)
다음 단계: /green — 테스트를 통과하는 최소 구현을 작성합니다
```

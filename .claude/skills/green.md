# /green — 최소 구현 스킬

## 역할
TDD 사이클의 **Green 단계**: 테스트를 통과하는 최소한의 코드를 작성한다.
"동작하는 코드"가 목표이며, "아름다운 코드"는 다음 단계(/refactor)의 몫이다.

## 규칙
- **테스트 파일은 수정하지 않는다**
- 테스트가 통과하면 **즉시 멈춘다** — 더 이상 코드를 추가하지 않는다
- 리팩토링하지 않는다 (구조 개선, 추상화, 변수명 개선 모두 금지)
- 하드코딩이라도 테스트가 통과하면 일단 OK

## 왜 최소 구현인가?

> "Make it work, make it right, make it fast"
> — Kent Beck

Green 단계에서 완벽한 코드를 쓰려 하면:
- 과도한 추상화가 생긴다
- 요구사항을 벗어난 코드가 추가된다
- 리팩토링할 때 변경 범위가 불명확해진다

## 실행 절차

1. 현재 실패하는 테스트 목록을 확인한다
2. **첫 번째 실패 테스트**를 통과시키는 최소 코드를 작성한다
3. 테스트를 실행하여 통과를 확인한다
4. 모든 테스트가 통과할 때까지 2-3을 반복한다
5. 전체 테스트 통과를 확인하고 **멈춘다**

## 완료 신호
모든 테스트가 통과하면:
```
🟢 Green 완료
통과한 테스트: N개
다음 단계: /refactor — 동작을 유지하면서 코드를 개선합니다
```

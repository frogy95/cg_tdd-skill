# CLAUDE.md에서 개발 프로세스 문서 분리

## Context
현재 CLAUDE.md에 핵심 원칙 + 파이프라인 상세가 모두 포함되어 있다 (55줄).
CLAUDE.md는 매 대화마다 컨텍스트에 로드되므로, 프로세스 상세는 별도 문서로 분리하고 필요할 때만 참조하도록 개선한다.

## Step 1: 프로세스 문서 생성
**파일**: `docs/development-process.md`

현재 CLAUDE.md의 내용을 그대로 이동 + 보강:
- 핵심 원칙 4가지 상세 (의도 우선, 한 턴 한 책임, 하네스 엔지니어링, 스킬 기반 파이프라인)
- 파이프라인 스킬 요약 테이블
- 각 단계 간 입출력 관계 추가 (어떤 산출물이 다음 단계의 입력이 되는지)
- 파이프라인 흐름 다이어그램 추가

## Step 2: CLAUDE.md 경량화
**파일**: `CLAUDE.md`

```markdown
# choiji-guide-3

AI 네이티브 개발 프로세스를 정의하는 프로젝트.

## 개발 프로세스
기능 개발 시 TDD 파이프라인을 따른다: `/spec → /red → /green → /refactor → /verify → /commit`
상세 원칙과 프로세스: @docs/development-process.md

## 언어 규칙
- 응답 언어: 한국어
- 코드 주석: 한국어
- 커밋 메시지: 한국어
- 변수명/함수명: 영어 (코드 표준 준수)
```

핵심: 파이프라인 커맨드 한 줄 + `@` 링크만 남기고, 원칙 상세와 테이블은 모두 프로세스 문서로 이동.

## 파일 변경 목록
| 파일 | 작업 |
|------|------|
| `docs/development-process.md` | 신규 생성 — 원칙 + 파이프라인 상세 |
| `CLAUDE.md` | 경량화 — 55줄 → ~10줄 |

## 검증
1. CLAUDE.md가 10줄 이내로 간결해졌는지 확인
2. `docs/development-process.md`에 기존 내용이 빠짐없이 포함되었는지 확인
3. `/spec` 호출 시 프로세스 문서를 자연스럽게 참조할 수 있는지 확인
